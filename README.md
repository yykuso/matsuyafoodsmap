# matsuyafoodsmap

松屋フーズグループの店舗情報を地図で可視化し、ブランド別や店舗限定メニュー実施店舗で絞り込めるWebアプリです。

## 主な機能

- 全国の店舗を地図上にプロットして表示
- ブランドごとに色分けした店舗表示
- ブランドチェックボックスによる絞り込み
- 店舗限定メニューCSVによる絞り込み
- 実施中/過去メニューを選択して対象店舗のみ表示
- 店舗クリックでポップアップ表示
- 公式サイト（ブランド別）/Googleマップ/Appleマップリンクをポップアップに表示
- 同一座標の店舗が重なりにくいように座標オフセット
- 地図の中心座標とズームをローカル保存して次回復元
- ブランド/店舗限定フィルターの選択状態をローカル保存して次回復元
- 現在地ボタン、ズーム操作、スケール表示
- スマートフォン向けレイアウト対応

## 使い方

1. ローカルサーバーでプロジェクトを配信します。
2. ブラウザでトップページを開きます。
3. 右上の「ブランド」チェックボックスと「店舗限定」セレクトで表示対象を絞り込みます。
4. 店舗をクリックして外部地図アプリへの検索リンクを利用します。

ローカル起動例:

```bash
python -m http.server 8000
```

## データ構成

- `data/matsuyafoods.csv`
	- 全店舗の基本データ
	- 主な列: `code`, `name`, `main`, `brand`, `bname`, `longitude`, `latitude`
	- `main`
		- `1`: メイン側店舗（地図描画の基本行）
		- `0`: 併設側店舗（表示名・公式リンク補完に使用）
	- `brand`（10進桁フラグ）
		- 1の位: 松屋
		- 10の位: 松のや
		- 100の位: マイカリー食堂
		- 1000の位: その他ブランド群
	- `bname`
		- `brand=1000` 系のブランド名（色分けとラベルに使用）
		- 旧フォーマット互換のため、未設定行は座標ずれ補正を実施
- `data/*.csv`（キャンペーン/店舗限定メニュー）
	- 1列目に店舗コードを持つCSV
	- 選択したCSVの店舗コードに一致する店舗のみを表示

## ロジック概要

### 1. CSV正規化

- `js/common.js` の `csvToGeoJSON()` で、CSVを一旦正規化してからGeoJSON化
- `bname` 列が欠ける旧フォーマット行は、`longitude/latitude` のずれを補正して吸収

### 2. 併設店の扱い

- 同一座標（小数6桁）+ 同一 `brand_code` を1グループとして扱う
- 地図に直接表示する点は `main != 0` 行を採用
- 同時に、同グループの全ブランド側情報を `sub_store_labels` に保持
	- 例: `matsuya`, `matsunoya`, `mycurry`, `other` ごとの `code/name/color`

### 3. ブランドフィルター

- チェックボックス未選択: 全表示
- 1つ以上選択: 選択したブランドフラグをすべて満たす店舗のみ表示（AND）
- 表示名は選択ブランドに応じて `sub_store_labels` 側へ差し替える場合がある
	- 目的: 併設店で、選択中ブランド側の店名を優先表示する

### 4. ポップアップと公式サイトリンク

- クリック時の `e.features[0]` ではプロパティが省略されるケースがあるため、
	`currentDisplayedGeoJSON` からコード一致で完全データを再取得
- 公式サイトリンクは「主店舗 + 併設側ブランド」をブランド名付きで縦並び表示
- リンク先は
	`https://pkg.navitime.co.jp/matsuyafoods/spot/detail?code=<code>`
	を使用
- 並び順は `松屋 -> 松のや -> マイカリー食堂 -> その他` を優先
